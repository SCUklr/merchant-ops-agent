# merchant-ops-agent 项目技术文档（框架版）

## 1. 文档目标与范围

本文档用于快速统一项目技术认知，先提供框架，后续按迭代逐步补全实现细节与指标数据。

当前版本聚焦：
- 业务场景与目标
- 功能模块拆解
- 端到端流程（检索 - 分析 - 执行）
- 建议文件结构与模块职责

---

## 2. 业务场景与目标

### 2.1 业务场景
- 电商商家在日常运营中存在大量高频问题：商品规则咨询、活动 SOP 查询、订单异常定位、评价归因分析、操作建议生成。
- 传统处理方式依赖人工查资料与跨系统操作，链路长、响应慢、结果一致性不足。

### 2.2 项目目标
- 将分散问题转化为标准化闭环：`用户提问 -> 知识检索 -> 任务分析 -> 工具执行 -> 结果校验 -> 输出建议`。
- 在保证正确性和可解释性的前提下，显著降低问题处理耗时。

---

## 3. 功能框架（V1）

### 3.1 对话与意图识别
- 识别用户问题类型（规则问答 / 数据查询 / 异常定位 / 操作执行）。
- 生成结构化任务上下文，供后续编排链路使用。

### 3.2 RAG 知识检索
- 接入规则、SOP、FAQ、案例等知识源。
- 支持分块、召回、重排与证据引用。

### 3.3 Agent 任务编排
- 任务拆解、工具路由、结果聚合。
- 高风险任务启用规则兜底与失败回退。

### 3.4 业务工具层（Tools）
- 订单查询
- 商品查询
- 工单生成
- 通知推送

### 3.5 回答生成与校验
- 结合检索证据与工具返回结果生成回答。
- 增加规则校验，降低高风险错误输出。

### 3.6 评测与回测
- 构建 case 集，追踪命中率、成功率、延迟等关键指标。
- 支持 Prompt 与编排策略迭代对比。

---

## 4. 端到端流程（概览）

1. 用户输入问题  
2. 意图识别与任务拆解  
3. RAG 检索获取证据  
4. 按需调用业务工具  
5. 规则校验与风险兜底  
6. 输出可执行建议与依据
